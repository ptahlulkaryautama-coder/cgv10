# CGV10 Project Instructions

This is a premium website/app project built with Next.js.

Always use the ui-ux-pro-max skill for:
- UI design
- UX structure
- landing page design
- dashboard/app interface
- layout
- typography
- spacing
- responsive design
- accessibility
- visual hierarchy
- component design
- conversion-focused sections

Before editing code:
1. Inspect the project structure first.
2. Understand the existing stack.
3. Check existing files before creating new files.
4. Create or follow a design system.
5. Explain the implementation plan.
6. Then edit code.

Design direction:
- premium
- clean
- modern
- trustworthy
- responsive
- production-ready
- elegant but practical
- strong visual hierarchy
- clear call-to-action

Avoid:
- generic AI-looking design
- random gradients
- weak typography
- inconsistent spacing
- unclear CTA
- unnecessary animation
- messy layout
- oversized sections without purpose
- removing working files without reason

Technical rules:
- Use the existing Next.js App Router structure.
- Prefer Tailwind CSS for styling.
- Keep code clean and readable.
- Keep components reusable when appropriate.
- Do not over-engineer.
- Do not add unnecessary libraries unless needed.

After editing:
- run npm run lint if available
- run npm run build if needed
- summarize changed files
- explain the design rationale
