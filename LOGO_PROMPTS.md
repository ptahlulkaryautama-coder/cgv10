# Logo Direction and Prompt Kit

Approved public identity:

CGV10 — Portal Digital Warga  
Cipta Greenville • RT 010 / RW 021

Do not mention any city name in logo files, prompt outputs, brand previews, poster mockups, or supporting copy.

## Brand Architecture

CGV10 is the parent brand and the main resident portal.

PALUGADA CGV is a sub-brand for marketplace warga activity inside the CGV10 ecosystem.

Cipta Greenville and RT 010 / RW 021 are identity badges. They support context and trust, but they should not be merged into the main small logo lockup.

Feature icons and category icons are separate UI assets. They should not be embedded inside the CGV10 or PALUGADA CGV logo.

## A. CGV10 Main Logo Prompt

Use this prompt to generate the parent brand logo.

```text
Create a clean premium vector logo for "CGV10" as a digital resident portal brand.

Logo composition:
- Simple CGV10 wordmark
- Small companion icon combining community, house, and trust
- Icon should feel civic, neighborly, calm, and reliable
- Balanced horizontal lockup for website header use
- Also works as a compact mark for mobile, favicon, and print

Visual style:
- Premium, clean, modern, trustworthy
- Flat vector design
- Minimal geometric construction
- Rounded but not playful
- Strong readability at small sizes

Color palette:
- Deep green as the primary brand color
- Warm gold as a restrained trust accent
- Cream or warm off-white as the supporting background
- Avoid harsh neon colors and random gradients

Must include:
- The wordmark text: CGV10
- A simple icon that suggests community, house, and trust
- Clear spacing between icon and wordmark
- Scalable SVG-ready shapes

Must avoid:
- No complex emblem
- No 3D
- No tiny text
- No slogan inside the logo
- No city name
- No overloaded symbols
- No too many icons
- No official-seal style unless used only as a separate poster emblem
- No detailed illustration
```

### CGV10 Negative Prompt

```text
Avoid crowded emblem, government seal, tiny lettering, skyline, city name, 3D render, glossy mockup, mascot, photo texture, complex illustration, excessive gradients, too many icons, slogan text, long descriptor text, unreadable details, ornate badge, clipart style.
```

### CGV10 Output Checklist

- Readable at website header size.
- Clear when reduced to favicon or app icon scale.
- Works in full color, one color, and reversed color.
- Separates the CGV10 logo from identity badges.
- Does not include all portal features inside one mark.

## B. PALUGADA CGV Logo Prompt

Use this prompt to generate the marketplace sub-brand logo.

```text
Create a clean premium sub-brand logo for "PALUGADA CGV" with the descriptor "Marketplace Warga".

Logo composition:
- Simple wordmark: PALUGADA CGV
- Optional descriptor below or beside the wordmark: Marketplace Warga
- Small icon concept combining a shopping cart with community or local neighborhood cues
- Friendly marketplace feel while still matching a premium resident portal ecosystem
- Designed as a sub-brand under CGV10, not as a separate unrelated company

Visual style:
- Clean flat vector logo
- Simple, friendly, readable
- Premium but approachable
- Consistent with the CGV10 green, warm gold, and cream brand palette
- Suitable for website section headers, marketplace pages, social preview, and print

Color palette:
- Deep green as the main color
- Warm gold for small accents
- Cream or warm off-white for background use

Must include:
- The wordmark text: PALUGADA CGV
- The descriptor: Marketplace Warga
- A simple cart plus community or neighborhood icon concept
- Clear spacing and simple SVG-ready shapes

Must avoid:
- No overloaded category icons inside the logo
- No 3D
- No tiny text
- No crowded composition
- No city name
- No product collage
- No complex emblem
- No unrelated e-commerce style that feels detached from warga/community context
```

### PALUGADA CGV Negative Prompt

```text
Avoid crowded marketplace collage, multiple category icons inside the logo, tiny descriptor text, 3D shopping cart, glossy render, neon sale badge, city name, complex emblem, product photos, excessive gradients, clipart, playful cartoon mascot, unreadable detail.
```

### PALUGADA CGV Output Checklist

- Reads clearly as PALUGADA CGV.
- Descriptor "Marketplace Warga" is legible in larger placements and optional in very small placements.
- Feels connected to CGV10 through color, shape language, and restraint.
- Does not replace the CGV10 parent brand.
- Does not include all marketplace categories inside one logo.

## C. CGV10 Feature Icon Set Prompt

Use this prompt for the main resident portal feature icons.

Icons:

1. Warga
2. Layanan
3. Keuangan
4. Informasi
5. Keamanan

```text
Create a consistent SVG-ready line icon set for the CGV10 resident portal.

Icon names:
- Warga
- Layanan
- Keuangan
- Informasi
- Keamanan

Direction:
- Simple line icons
- Consistent stroke width
- Rounded line caps and joins
- Deep green line work with restrained warm gold accents
- Clear at small UI sizes
- Friendly, trustworthy, and practical
- Built for navigation cards, section labels, and dashboard-style UI

Icon concepts:
- Warga: residents, people, or community
- Layanan: service, assistance, or document/help flow
- Keuangan: transparent finance, receipt, wallet, or chart
- Informasi: announcement, notice, or information panel
- Keamanan: shield, gate, or security check

Must avoid:
- No 3D
- No detailed illustration
- No tiny text
- No emoji style

- No decorative badge background that reduces clarity
```

### Feature Icon Output Checklist

- All icons share the same grid, stroke width, corner radius, and visual weight.
- Each icon remains recognizable at 24 px and 32 px.
- Icons can work in one color if needed.
- Icons stay separate from the CGV10 logo.

## E. Usage Guidance

Use the simplified CGV10 logo in the website header.

Use the PALUGADA CGV logo only on PALUGADA page/section placements, marketplace-related previews, or supporting marketplace material.

Use complex pengurus logo concepts only as reference, emblem, ceremonial, or poster direction. Do not use complex concepts as small header logos because detailed shapes and tiny text become unreadable.

Do not put all resident features, marketplace categories, identity badges, and slogans inside one logo. Separate the system into parent brand, sub-brand, feature icons, category icons, and identity badges.

Keep logos scalable and readable. Every final logo should work in:

- Full-color horizontal lockup
- Compact icon-only mark
- One-color version
- Reversed version on dark green
- Favicon or app icon crop
- Print-safe version

## Recommended File Mapping

Use stable filenames so the website can swap approved artwork later without code changes.

- CGV10 mark: `public/assets/brand/cgv10-mark.svg`
- CGV10 wordmark: `public/assets/brand/cgv10-wordmark.svg`
- PALUGADA mark: `public/assets/brand/palugada-mark.svg`
- PALUGADA wordmark: `public/assets/brand/palugada-wordmark.svg`
- CGV10 feature icons: `public/assets/icons/warga.svg`, `layanan.svg`, `keuangan.svg`, `informasi.svg`, `keamanan.svg`
- PALUGADA category icons: `public/assets/icons/barang.svg`, `kuliner.svg`, `jasa.svg`, `properti.svg`, `lainnya.svg`

## Final Review Rules

Before approving any generated logo or icon:

1. Check whether it is readable at small size.
2. Remove any city name or unapproved location detail.
3. Remove tiny text and slogans from logo marks.
4. Confirm feature icons are not embedded in the logo.
5. Confirm category icons are not embedded in the marketplace logo.
6. Confirm the visual tone is premium, clean, modern, trustworthy, and practical.
