# CGV10 Project Brief

## Project Name

CGV10

## Project Type

Premium community portal website and future web app / PWA.

## Full Identity

CGV10 is the digital citizen portal for Cipta Greenville RT 010 / RW 021.

## Location / Display Identity
Use this public identity only:

CGV10 — Portal Digital Warga  
Cipta Greenville • RT 010 / RW 021

Do not mention Jakarta Selatan.
Do not mention any city unless explicitly approved later.
If location detail is needed, use only "Cipta Greenville RT 010 / RW 021".

## Core Purpose

CGV10 is designed as an integrated digital ecosystem for residents of Cipta Greenville. The platform connects community communication, resident services, financial transparency, local marketplace activity, security information, and administrative services in one clean, trusted, and easy-to-use interface.

## Target Users

1. Residents of Cipta Greenville.
2. RT/RW administrators.
3. Treasurer / bendahara.
4. Secretary / sekretaris.
5. Security team / satpam.
6. Facility managers.
7. Local resident sellers / UMKM.
8. Super admin / technical administrator.

## Primary Goals

1. Help residents access community information quickly.
2. Make RT cash and monthly contribution information more transparent.
3. Centralize announcements, events, services, and emergency information.
4. Support local resident businesses through Palugada CGV marketplace.
5. Give RT administrators a more organized system for data, payments, announcements, and reports.
6. Prepare the foundation for a future resident dashboard, admin dashboard, and PWA.

## Main Modules

1. Resident management.
2. Announcements and community news.
3. Monthly dues and finance transparency.
4. Resident services.
5. Palugada CGV local marketplace.
6. Security and guest access.
7. Facility booking.
8. Letters and administration.
9. Reports and statistics.
10. Important contact directory.
11. Polling and voting.
12. Donations and community charity.

## MVP v1 Focus

For the first public version, focus on a premium landing page / portal homepage, not a full functional app yet.

The landing page should communicate:

1. What CGV10 is.
2. Why residents need it.
3. What problems it solves.
4. What modules are planned.
5. What features are included in MVP v1.
6. How residents and administrators benefit.
7. How the community can support or join the project.

## Suggested First Page Sections

1. Hero section: CGV10 as the digital portal for Cipta Greenville residents.
2. Community value proposition.
3. Resident dashboard preview.
4. Core modules overview.
5. Finance transparency and iuran section.
6. Palugada CGV marketplace section.
7. Security and emergency readiness section.
8. Admin / pengurus dashboard section.
9. MVP roadmap.
10. Call-to-action for residents and pengurus.

## Visual Direction

Premium but friendly community technology.

The design should feel:

* Clean
* Trustworthy
* Warm
* Organized
* Modern
* Civic/community-oriented
* Easy for all age groups
* Professional but not corporate-cold

## Color Direction

Use a premium green-based civic palette:

* Deep community green
* Soft cream background
* Warm gold accent
* White cards
* Muted gray text
* Subtle borders

Avoid overly flashy colors, random gradients, cheap shadows, and overly futuristic UI.

## Typography Direction

Use clear, highly readable typography.

The design should prioritize:

* Strong headline hierarchy
* Easy scanning
* Comfortable reading
* Large enough mobile text
* Professional dashboard-style labels

## UX Principles

1. Mobile-first.
2. Resident-friendly.
3. Clear navigation.
4. Simple language.
5. High trust and transparency.
6. No unnecessary animation.
7. Strong accessibility.
8. Easy CTA.
9. Clear distinction between public portal and future member/admin areas.

## Technical Direction

Use the existing Next.js App Router project.

Preferred stack:

* Next.js
* TypeScript
* Tailwind CSS
* Responsive single-page landing page first
* Future-ready structure for PWA and dashboard

## Build Rules

1. Do not build the full app yet.
2. Do not create login/auth/database yet.
3. First build only the premium public-facing landing page.
4. Keep code clean and maintainable.
5. Use existing project structure unless there is a clear reason to add components.
6. Run npm.cmd run lint and npm.cmd run build after implementation.

## Avoid

* Generic SaaS template feel.
* Overly corporate language.
* Heavy glassmorphism.
* Random AI-style gradients.
* Too much animation.
* Too many fake metrics.
* Full backend/app features before the landing page is approved.
