# CGV10 Phase 1 Scope Lock

Approved public identity:

CGV10 — Portal Digital Warga  
Cipta Greenville • RT 010 / RW 021

This document locks the current product direction after the completed Task 1 to Task 17 work. The existing static multi-page portal, brand structure, deployment notes, placeholder content rules, and public asset direction remain valid.

## Product Direction

CGV10 should be treated as a phased resident portal product, not a single all-at-once app build.

Phase 1 is the current focus:

Public resident portal for Cipta Greenville RT 010 / RW 021 with PALUGADA CGV as a priority campaign.

The website remains a static public prototype. It should communicate structure, trust, and future readiness without implying that private workflows or live transactions are already active.

## Phase 1 Scope

Phase 1 includes:

- Public resident portal
- Pengurus information structure
- Keuangan / kas transparency structure
- Pengumuman structure
- Kontak penting structure
- PALUGADA CGV campaign and catalogue preview
- Roadmap context for future product evolution

Phase 1 does not include:

- Login
- Auth
- Database
- Backend
- API routes
- Payment
- Checkout
- Cart
- Transaction system
- Real complaint system
- Admin dashboard
- Group chat
- Push notification
- Real app functionality

## PALUGADA Priority Campaign

PALUGADA CGV is a priority campaign from the elected RT direction. It should be treated as an important early public-facing module inside CGV10, not as a separate parent brand.

In Phase 1, PALUGADA CGV should be positioned as:

- Marketplace Warga
- A resident promotion and directory space
- A catalogue preview for warga offerings
- A campaign that helps residents understand the future marketplace direction

PALUGADA CGV should not look like an active transaction system yet.

## PALUGADA Phase 1 Content Model

Categories:

- Lainnya

Item card fields:

## Roadmap Context

Keep the full roadmap as future context:

1. Phase 1 — Public portal + PALUGADA campaign
2. Phase 2 — Login warga + profil rumah
3. Phase 3 — Community feed
4. Phase 4 — PALUGADA katalog
5. Phase 5 — Keluhan, lapor RT, security

Only Phase 1 should be presented as the active public prototype scope. Later phases should be framed as future direction, not active functionality.

- Do not invent names.
